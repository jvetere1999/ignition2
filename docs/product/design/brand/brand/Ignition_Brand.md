# Ignition Brand Guidelines

## Product Name

**Ignition**

## Domain

**https://ignitions.app**

---

## Positioning

Ignition is a *starter engine*.
It helps users **begin** with minimal friction and build momentum naturally.

### What Ignition Is

- A starter engine for focus, movement, and learning
- Momentum-oriented
- Calm and direct
- Non-guilting

### What Ignition Is NOT

- Not a planner
- Not a habit tracker
- Not a "do more" system

---

## Brand Voice

### Core Principles

- Calm
- Direct
- Non-clinical
- Non-guilting
- Momentum-oriented
- Short sentences
- Neutral encouragement (never hype)

### Tone Rules

- Never shame
- Never say "optimize", "maximize", "crush", "win"
- Never imply failure for inactivity
- Avoid future-pressure language

### Allowed Phrases

- "Start small."
- "Good start."
- "Pick one thing."
- "Ready when you are."
- "Let's begin."
- "Underway."
- "Welcome back."

### Disallowed Language

- "Productivity"
- "Discipline"
- "Consistency"
- "Goals" (unless referring to the existing feature)
- "Success"
- "Optimize"
- "Maximize"
- "Crush it"
- "Win"

---

## Visual Identity

### Color Direction

- Dark-first UI (existing themes preserved)
- Accent color: **Ignition Amber**
  - HEX: `#FFB703`
  - Used sparingly for CTAs, focus rings, progress states
- Secondary neutral: cool gray / slate (existing palette acceptable)

### Typography

- Preserve existing font stack
- Emphasize hierarchy through weight and spacing, not color

---

## SVG Icon System

### Style Rules

- SVG only (no PNG for icons)
- Stroke-based icons
- Stroke width: `1.75`
- Rounded line caps and joins
- No fills unless explicitly needed
- ViewBox: `0 0 24 24`
- CurrentColor for stroke

### Brand Assets

Located in `src/assets/brand/`:

| File | Purpose |
|------|---------|
| `ignition-mark.svg` | Primary brand mark (spark icon) |
| `ignition-mark-mono.svg` | Monochrome version |
| `ignition-wordmark.svg` | Full wordmark with icon |

### Usage Guidelines

- Use mark alone for favicon, app icon, small contexts
- Use wordmark for header, marketing, larger contexts
- Mark should work at 16px, 24px, 32px, 64px
- Always maintain stroke proportions

## Technical Notes

- Database tables
- Environment variable prefixes
- Cookies
- API routes

### DO Rename

- UI copy
- Page titles
- Metadata
- Public-facing strings
- Documentation references
