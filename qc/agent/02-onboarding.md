# Onboarding Tests

## Test 1: First-Time User Experience

```
Go to [URL] in an incognito window. Create a new account or log in for the first time.

1. Complete authentication
2. Observe what happens immediately after first login
3. Is there an onboarding flow, tutorial, or welcome screen?
4. If there's a multi-step onboarding, go through each step
5. Note what information is collected (name, preferences, goals, etc.)

Report:
- Describe the onboarding flow step by step
- How many steps/screens are there?
- Is it skippable?
- Does it feel welcoming and not overwhelming?
- Any confusing parts?
```

## Test 2: Profile Setup

```
As a new user on [URL], complete any profile setup.

1. Look for profile/account settings
2. Try to set a display name
3. Try to upload or change an avatar (if available)
4. Look for any personalization options
5. Save changes and verify they persist

Report:
- What profile fields are available?
- Did save work correctly?
- Were there validation errors for any fields?
- Is the UX intuitive?
```

## Test 3: Onboarding Skip & Resume

```
Go to [URL] as a new user.

1. Start the onboarding flow
2. If there's a skip option, use it
3. Navigate around the app
4. Check if there's a way to resume/complete onboarding later
5. Look in settings or profile for "complete setup" prompts

Report:
- Can onboarding be skipped?
- Is there a reminder to complete it?
- Does the app function normally after skipping?
```

## Test 4: Preference Collection

```
During or after onboarding on [URL]:

1. Look for preference settings (themes, notifications, goals)
2. Set some preferences
3. Verify they take effect immediately
4. Log out and log back in
5. Verify preferences persisted

Report:
- What preferences are available?
- Do they apply immediately?
- Did they persist across sessions?
```
