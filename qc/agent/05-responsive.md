# Responsive Design Tests

## Test 1: Desktop (1920x1080)

```
View [URL] at 1920x1080 resolution.

1. Check if content uses the full width appropriately
2. Look for any awkward spacing or stretched elements
3. Verify text is readable
4. Check image quality

Report:
- Does the layout use space well?
- Any layout issues at large screen?
- Is content centered or full-width?
```

## Test 2: Laptop (1366x768)

```
View [URL] at 1366x768 resolution.

1. Check all content is visible without horizontal scroll
2. Verify navigation is usable
3. Check for any truncated text
4. Verify modals/dialogs fit on screen

Report:
- Does everything fit?
- Any horizontal scrolling?
- Text truncation issues?
```

## Test 3: Tablet Portrait (768x1024)

```
View [URL] at 768x1024 (iPad portrait).

1. Check if navigation adapts (sidebar collapse, hamburger menu)
2. Verify touch targets are adequate size (44px minimum)
3. Check form inputs are usable
4. Verify no content is cut off

Report:
- How does navigation adapt?
- Are touch targets large enough?
- Any layout breaks?
```

## Test 4: Mobile (375x812)

```
View [URL] at 375x812 (iPhone X/11/12 size).

1. Check all text is readable without zooming
2. Verify buttons are tappable (not too small or close together)
3. Check if horizontal scroll exists (it shouldn't)
4. Test any forms - are inputs full width?
5. Verify images scale correctly

Report:
- Is text readable?
- Are buttons tappable?
- Any horizontal overflow?
- Form usability on mobile?
```

## Test 5: Small Mobile (320x568)

```
View [URL] at 320x568 (iPhone SE/small Android).

1. Check for critical layout breaks
2. Verify core functionality is accessible
3. Check navigation still works
4. Note any elements that don't fit

Report:
- Does the app work on very small screens?
- What breaks, if anything?
- Is core functionality preserved?
```
